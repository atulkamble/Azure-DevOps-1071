# 📌 Project Overview

This project demonstrates:

* ✅ AKS Cluster creation using Azure CLI
* ✅ Authentication & kubeconfig setup
* ✅ Deployment of NGINX application on Kubernetes
* ✅ Exposure via LoadBalancer service
* ✅ Access application via public IP

---

# 🧱 Architecture

```
Azure CLI
   ↓
Resource Group (devops)
   ↓
AKS Cluster (mycluster)
   ↓
Kubernetes Nodes (2 nodes)
   ↓
Deployment (NGINX Pods)
   ↓
Service (LoadBalancer)
   ↓
Public IP → Browser Access
```

# ⚙️ Prerequisites

* Azure Account
* Azure CLI installed
* kubectl installed
* Git installed
* VS Code (optional)

---

# 🔐 Step 1: Login to Azure

```bash
az login
```

---

# 📦 Step 2: Create Resource Group

```bash
az group create --name devops --location eastus
```

---

# ☸️ Step 3: Create AKS Cluster

```bash
az aks create \
  --resource-group devops \
  --name mycluster \
  --node-count 2 \
  --enable-managed-identity \
  --generate-ssh-keys
```

### 🔍 Key Parameters

| Parameter                 | Description                 |
| ------------------------- | --------------------------- |
| --node-count              | Number of worker nodes      |
| --enable-managed-identity | Secure identity for cluster |
| --generate-ssh-keys       | Auto-generate SSH keys      |

---

# 🔗 Step 4: Connect to AKS Cluster

```bash
az aks get-credentials \
  --resource-group devops \
  --name mycluster \
  --overwrite-existing
```

---

# ✅ Step 5: Verify Cluster

```bash
kubectl get nodes
```

✔️ Output should show **2 Ready nodes**

---

# 📥 Step 6: Clone Application Repository

```bash
cd Downloads
git clone https://github.com/atulkamble/nginx-k8s-project.git
cd nginx-k8s-project
code .
```

---

# 🚀 Step 7: Deploy Application

```bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
```

---

# 🔎 Step 8: Verify Deployment

# 🧠 Important Points (Interview + Real World)

* AKS uses **managed control plane** (no need to manage master nodes)
* LoadBalancer service automatically provisions **Azure Public IP**
* Managed Identity improves **security (no secrets)**
* Always verify using:

  * `kubectl get nodes`
  * `kubectl get pods`
  * `kubectl get svc`
* Production setups use:

  * Azure CNI / Kubenet networking
  * Ingress Controller instead of LoadBalancer
  * Azure Monitor / Prometheus for observability
